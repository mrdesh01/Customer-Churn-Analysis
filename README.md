# 📊 GP Customer Churn Analysis

This project presents a comprehensive data analysis of **customer churn behavior** for Grameenphone — a leading telecom operator in Bangladesh. The objective is to uncover insights from customer data that explain why users leave the service and identify at-risk customer segments to inform strategic retention efforts.

---

## 🧠 Key Objectives

- Analyze customer churn trends based on recharge, usage, and package types.
- Segment customers by tenure and observe behavioral differences.
- Visualize important metrics to draw actionable business insights.

---

## 📁 Dataset

The dataset includes various customer-level features:
- `monthly_recharge`
- `data_usage_gb`
- `package`
- `segment_tenure`
- `churn` (target variable)

> 📌 *Note: The data was analyzed using Python libraries like Pandas, Seaborn, and Matplotlib.*

---

## 🛠️ Tools & Technologies

- **Python** (Pandas, NumPy, Matplotlib, Seaborn)
- **Jupyter Notebook**
- **Data Visualization**
- **Exploratory Data Analysis (EDA)**

---

## 📈 Visualizations Used

| Chart Type   | Purpose                                |
|--------------|-----------------------------------------|
| Pie Chart    | Churn percentage distribution           |
| Histogram    | Monthly recharge behavior               |
| Boxplot      | Data usage distribution                 |
| Barplot      | Package-wise customer distribution      |
| Countplot    | Segment-wise churn comparison           |

---

## 🔍 Key Findings

- 📉 **Churn Rate:** A noticeable portion of customers are leaving.
- 💸 **Low Recharge & Usage = High Risk:** Low spenders are more likely to churn.
- ⏳ **Tenure Matters:** New customers show higher churn probability.
- 📦 **Package Effectiveness:** Some packages retain users better than others.

---

## ✅ Conclusion

This analysis provides deep insights into customer churn and sets the foundation for:
- Predictive modeling (e.g., churn prediction using ML)
- Customer segmentation
- Data-driven retention campaigns
